¿Cómo agregar un evento?

// sin backend · sin base de datos · solo GitHub

PASO 01

Abre un Issue

Ve al repositorio en GitHub y abre un nuevo issue usando el template event.md. Llena los campos del formulario.

PASO 02

GitHub Actions

Un workflow automático se dispara, parsea el issue y regenera el archivo events.json con los datos del evento.

PASO 03

Deploy automático

GitHub Pages se actualiza automáticamente y el evento aparece en el sitio en minutos. Sin intervención manual.

PASO 04

Comunidad revisa

Los maintainers pueden aprobar, cerrar o etiquetar issues. Solo eventos con label approved se publican.
